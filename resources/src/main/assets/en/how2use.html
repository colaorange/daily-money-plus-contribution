Concept

The concept of daily money is very simple
  1. Different accounts (you create accounts depends on your case)
  2. Money transaction between accounts. (you record record for each transaction)

Accounts

There are 5 types account, you can create/edit/delete it in Account Manager page.

  1. Income : salary etc..
  2. Expense : food, entertainment etc.
  3. Asset : cash, bank
  4. Liability : credit card, loan.
  5. Other : ...like a asset but is not asset.
* To sort your account, you just add 1,2,.. or A.B,.. as the prefix of name of account.
* Account name could have a hierarchical by naming with .(dot), Ex: Food.lunch, Food.dinner . When editing transaction record, it provides better selecting of account.

The transaction record

When you spent money then you can click the Add Record on the Main page to create a new record. for example you spent $10 bought a meal, in Record Editor page, you

  1. Select from account : Cash
  2. Select to account : Food
  3. Select date : today
  4. Input money by calculator : 10
  5. Input description : hot dog, coke.
  6. click Create then click Close if no more transaction needs to be created.
this is a just very common use case of daily-money, but, what if your salary is deposit to bank directly, you use credit card buy books on internet? daily-money provide different type of account s, and you could create various accounts against 5 account type.

Balance Reports

Ranged Balance

In ranged balance reports, I count balance of all the details that you created. The balance is calculated in a time range (current week, current month, current year, or you can change to other range by click the arrow button)

Cumulated Balance

To calculate cumulated balance (ex, how much cash I have now, how much liability I have now. not how much I spent this week), you have to set a initial value to a account(generally Asset or Liability type accounts have non-zero initial value. Income and Expense type account should have no initial value).
If you forgot to set initial value of an account, there is a simple way to adjust it. for example , "I have cash 333, but cumulated balance reports shows I have cash -100)
  1. Edit cash account
  2. Set initial value of cash to 433, result of (333 - (-100))

Multiple Books

You can create multiple-books and active one book as the 'Working-Book'. You can define a currency symbol (ex, '$','Â¥'..etc) and the position of the symbol of a book. After you created a book, application will allocate another database to store accounts and details, data between books are not able to be transfer since they are isolated.

When exporting current data to CSV, it will export two files, details.csv and accounts.csv, which are sharing between all books, and another two files,details-'id'.cvs and accounts-'id'.csv, 'id' is the book id),for working book only, When importing data from CSV, you can choose to import from shared or non-shared file.
